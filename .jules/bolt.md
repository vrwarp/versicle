## 2025-06-15 - Function Call Hoisting in Selectors
- **Bottleneck:** In `useAllBooks` inside `src/store/selectors.ts`, `getDeviceId()` was being called synchronously on every iteration of a hot path loop (mapping over all books when the reading progress updates). Although `getDeviceId()` internally caches the result in a variable, the function invocation itself within an O(N) loop mapping hundreds/thousands of objects still carries overhead on every page turn.
- **Solution:** Hoisted the `getDeviceId()` function call outside of the `.map()` loop, storing its result in `currentDeviceId` and passing the string directly into `resolveProgress()`.
- **Learning:** Even when a fast-path function internally caches its result, invoking it repeatedly inside an O(N) map loop during frequent React state updates (like page turns in reading mode) incurs unnecessary function call overhead. Hoist static or session-level values out of mapping loops inside `useMemo` blocks or selectors.
